# CLAUDE.md

이 저장소는 반도체 시료 생산주문관리 시스템 과제의 **MVC 콘솔 스켈레톤 PoC**다.
목표는 최종 프로젝트를 만들기 전에 콘솔 애플리케이션에서 Model, View, Controller 역할을
어떻게 나눌지 빠르게 검증하는 것이다.

## 1. 진행 방식

- 이 PoC는 별도 TDD 사이클 없이 빠르게 구현한다.
- 기능보다 구조 검증이 목적이므로, 과도한 기능 확장은 하지 않는다.
- 최종 프로젝트(`SampleOrderSystem-myungjunkim-23048751`)에서 참고하거나 복사해 사용할 수
  있는 형태로 단순하고 읽기 쉽게 작성한다.
- 구현 중 최종 프로젝트에 유용한 구조가 생기면 README 또는 문서에 재사용 방법을 남긴다.

## 2. 문서 역할

- `docs/PRD.md`: 이 PoC가 왜 필요한지, 어떤 사용 가치를 검증하는지 설명하는 제품 관점 문서.
- `docs/SPEC.md`: PoC가 제공해야 하는 기능, 화면 흐름, MVC 책임 분리 기준을 정의하는 명세서.
- `docs/PLAN.md`: 구현 순서, 예상 파일 구조, 검증 방법을 담은 실행 계획.

## 3. 구현 원칙

- Model은 데이터와 상태를 표현한다.
- View는 콘솔 출력과 입력 안내를 담당한다.
- Controller는 사용자 입력을 받아 Model과 View를 연결한다.
- 도메인은 반도체 시료 주문 시스템과 맞추되, 저장소나 생산 로직까지 깊게 구현하지 않는다.
- `print`/`input` 호출은 View 또는 CLI 경계에 모은다.

## 4. 커밋 규칙

- 커밋 제목은 영어로 작성한다.
- PoC는 작은 단위보다 완성된 흐름 단위로 커밋해도 된다.
- 권장 태그:
  - `docs:` 문서 작성/수정
  - `feat:` PoC 기능 구현
  - `chore:` 실행 환경, 설정, 정리

예시:

```text
docs: define console mvc poc scope
feat: implement console mvc sample flow
```

## 5. 완료 기준

- 사용자가 콘솔 메뉴를 통해 샘플 목록 조회 또는 등록 흐름을 따라갈 수 있다.
- 코드 구조만 봐도 Model/View/Controller 책임이 구분된다.
- 최종 프로젝트에서 어떤 구조를 가져갈 수 있는지 문서로 설명되어 있다.
