# PRD: Console MVC PoC

## 1. 목적

반도체 시료 생산주문관리 시스템은 콘솔 기반으로 동작한다. 최종 프로젝트를 바로 만들기 전에,
사용자 입력과 화면 출력, 상태 변경 로직이 뒤섞이지 않도록 MVC 구조를 작은 예제로 검증한다.

## 2. 대상 사용자

- 콘솔 시스템을 구현하는 개발자
- 최종 프로젝트에서 화면 흐름과 도메인 로직을 분리하고 싶은 리뷰어

## 3. 사용자 가치

- 메뉴 흐름을 바꿔도 데이터 구조나 업무 로직이 흔들리지 않는다.
- 테스트하기 어려운 `input`/`print` 호출을 애플리케이션 핵심 로직에서 분리할 수 있다.
- 최종 시스템의 시료 관리, 주문 관리 화면을 만들 때 기본 구조로 재사용할 수 있다.

## 4. 대표 시나리오

1. 사용자가 콘솔 메뉴를 본다.
2. "시료 등록"을 선택한다.
3. View가 필요한 입력 항목을 안내한다.
4. Controller가 입력을 받아 Model에 반영한다.
5. View가 등록 결과와 현재 목록을 보여준다.

## 5. 범위

포함:

- 간단한 콘솔 메뉴
- Model, View, Controller 역할 분리
- 예시 도메인으로 시료 등록/조회 흐름

제외:

- 파일/DB 저장
- 실제 생산 라인 큐 처리
- 복잡한 주문 승인/출고 로직
- 엄격한 입력 검증

## 6. 성공 기준

- MVC 구조가 눈으로 확인 가능하다.
- 같은 Model을 다른 View에서도 사용할 수 있도록 결합도가 낮다.
- 최종 프로젝트에 구조를 가져갈 때 큰 수정 없이 이해할 수 있다.
