# SPEC: Console MVC PoC

## 1. 기능 명세

이 PoC는 다음 최소 흐름을 제공한다.

| 기능 | 설명 |
|---|---|
| 메인 메뉴 표시 | 사용 가능한 작업 목록을 콘솔에 출력한다. |
| 시료 등록 | 시료 ID, 이름 등 간단한 값을 입력받아 Model에 추가한다. |
| 시료 목록 조회 | 현재 등록된 시료 목록을 출력한다. |
| 종료 | 프로그램을 종료한다. |

## 2. MVC 책임

| 계층 | 책임 | 금지할 것 |
|---|---|---|
| Model | 시료 데이터와 상태 보관 | 콘솔 입출력 |
| View | 메뉴와 결과 출력, 입력 안내 | 비즈니스 판단 |
| Controller | 입력 해석, Model 변경, View 호출 | 데이터 저장 방식 직접 구현 |

## 3. 예시 흐름

```text
사용자
  -> View: 메뉴 확인
  -> Controller: "시료 등록" 선택
  -> View: 입력 항목 안내
  -> Controller: 입력값 수집
  -> Model: 시료 추가
  -> View: 등록 완료 메시지 출력
```

## 4. 데이터 예시

```json
{
  "sample_id": "SMP-001",
  "name": "Logic Sample",
  "average_production_time": 5,
  "yield_rate": 0.9
}
```

## 5. 제약

- 데이터는 메모리에만 보관해도 된다.
- 입력값 검증은 구조 확인에 필요한 수준만 적용한다.
- 최종 프로젝트에서 그대로 복사될 수 있도록 의존성을 최소화한다.
